## Public Support Model

The intended support flow after publication is:

- bug reports: GitHub issue templates
- feature requests: GitHub issue templates
- usage questions and open discussion: GitHub Discussions
- security problems: private reporting path described in [SECURITY.md](SECURITY.md)

## What To Include In A Support Request

- platform: web, app, or both
- workspace or module involved
- provider mode and model if AIAnalysis is involved
- reproducible steps
- expected versus actual behavior
- sanitized logs, screenshots, or backup snippets
- whether the issue affects only a local environment or multiple environments

## Before Public Launch

Until the repository is fully published:

- use the repository docs first
- keep sensitive issues private
- use the security path in [SECURITY.md](SECURITY.md) for anything involving credentials, installer integrity, or local data exposure
