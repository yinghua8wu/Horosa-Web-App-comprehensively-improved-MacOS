# Horosa 行为准则

English guide: [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md)

本项目正在按“正式开源发布”的标准整理。我们希望 Horosa 既是一个严肃、可信的技术项目，也是一个对协作者友好、可长期协作的社区空间。

## 我们的承诺

我们承诺让 Horosa 社区中的参与过程保持尊重、专业，并尽可能避免骚扰与压迫性体验，不因背景、经验水平、身份、研究方向或观点差异而区别对待。

我们尤其希望协作者做到：

- 有分歧时保持准确，而不是轻蔑
- 做 review 时坦诚但不羞辱他人
- 对私人盘例、用户数据、截图、日志和 API 凭据保持谨慎
- 遇到取舍时先解释理由，而不是把冲突升级

## 我们鼓励的行为

有助于形成健康项目氛围的行为包括：

- 提供清楚、可复现的 bug 报告
- 留下建设性且技术上具体的 review 意见
- 说明假设、风险和迁移影响
- 对初学者保持友善，但不降低质量标准
- 尊重领域专家、研究者和维护者的专业判断

## 不可接受的行为

- 骚扰、威胁、恐吓或人身攻击
- 侮辱、轻蔑、羞辱式语言
- 未经允许公开私人盘例、用户数据、API Key 或内部日志
- 虚报测试结果、版本状态、兼容性或安全情况
- 故意破坏协作、review 或事故处理流程

## 适用范围

本准则适用于：

- 仓库讨论
- Issue 与 Pull Request
- 代码评审线程
- 版本规划与文档评审
- Horosa 维护过程中使用的相关协作空间

## 维护者职责

维护者应当：

- 一致地执行本准则
- 在合理范围内保护举报者免受报复
- 当行为问题开始影响协作时尽早介入
- 优先采用清晰边界、降级处理和留痕决策

## 举报方式

当前行为问题联系路径如下：

- 主联系人：`maxwelldhx@gmail.com`
- 备份维护者路径：通过 [SUPPORT_ZH.md](SUPPORT_ZH.md) 中公开的仓库支持路径私下联系维护者，并明确标注为行为问题

## 处理方式

当行为违反本准则时，维护者可以删除评论、暂停 review 线程、关闭贡献，或限制继续参与。对严重或重复违规者，可以进一步取消协作权限。

## 说明

本文件借鉴了成熟开源社区常用的行为准则结构，并针对 Horosa 的产品属性、隐私要求和研究语境做了定制。
