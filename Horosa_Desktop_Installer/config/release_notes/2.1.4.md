- **OpenAI 推理模型可用**:`gpt-5.x` / `o1` / `o3` / `o4` 以前只回「模型未返回可用内容」;现在对这些系列省略 `temperature`、改用 `max_completion_tokens`,可正常对话,`gpt-4.1` 等既有模型不受影响。
- **OpenAI reasoning models work**: `gpt-5.x` / `o1` / `o3` / `o4` previously returned only empty content; the request now omits `temperature` and uses `max_completion_tokens` for these families, so they reply normally. Existing models like `gpt-4.1` are unchanged.
- **真实错误透传**:上游 provider 报错(参数不支持、401/404 等)不再被吞成笼统提示,会显示真正原因,便于自查配置。
- **Real upstream errors surfaced**: provider failures (unsupported parameters, 401/404, etc.) are no longer swallowed into a generic message; the actual reason is shown.
- **凭据不再外泄**:请求失败时 `Authorization` / `x-api-key` / `LocalIp` 等敏感头在错误信息与日志中脱敏,URL 里的 `?key=` 也不写进日志。
- **Credentials redacted**: on a failed request, sensitive headers (`Authorization` / `x-api-key` / `LocalIp`) are redacted from error messages and logs, and `?key=` query strings are stripped from logged URLs.
