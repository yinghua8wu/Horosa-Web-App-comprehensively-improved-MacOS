- **多供应商切换修复**:供应商卡片现在可点击、并有明确「设为当前」按钮,真正切换生效;在编辑器里切换 provider 类型会清空旧 API Key,避免存出「新 URL + 旧 Key」导致的 401。
- **Multi-provider switching fixed**: provider cards are now clickable with an explicit "set as current" button, so switching actually takes effect; changing the provider type in the editor clears the old API key to avoid a "new URL + old key" 401.
- **Gemini 直连修复**:不再为 Gemini 多加 `Authorization: Bearer`(消除 `ACCESS_TOKEN_TYPE_UNSUPPORTED`);自定义供应商可用 `authHeaderName`/`authPrefix` 自定义鉴权头,兼容要求非 Bearer 的官方原生 key。
- **Gemini direct connection fixed**: no longer attaches a redundant `Authorization: Bearer` for Gemini (eliminates `ACCESS_TOKEN_TYPE_UNSUPPORTED`); custom providers can override the auth header via `authHeaderName`/`authPrefix` for native keys that don't use Bearer.
- **发送更稳**:流式生成期间不再被回车/重复点击并发触发;「停止生成」与切换/删除对话会真正中止流;新消息自动滚动到底;请求超时尊重你配置的 `requestTimeoutMs`。
- **Steadier sending**: streaming can no longer be double-triggered by Enter/clicks; stop/switch/delete now actually aborts the stream; new messages auto-scroll; requests honor your configured `requestTimeoutMs`.
- **静默失败可见**:向量检索失败、资料分块失败不再悄悄降级,会给出提示。
- **Silent failures surfaced**: embedding-retrieval and material-chunking failures are now reported instead of silently degrading.
