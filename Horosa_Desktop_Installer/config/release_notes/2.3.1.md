- **Fix: post-update restart "stuck at 100% for a long time"**: after an in-app update, the first restart runs a full verification of the freshly-installed runtime (~30–60s). It now shows an **animated progress bar + a clear "verifying after update, please wait" hint** instead of looking frozen, and is sped up **while keeping the full verification** (backend-ready ~29s → ~14s). Two recurrence causes are also fixed: the update marker lingering when the first launch fails (which forced every later launch onto the slow path), and a stale pid-file from a previous run wrongly blocking startup.
- **Fix: AI analysis "stops after a few sentences"**: long replies (e.g. DeepSeek) no longer cut off mid-stream (a race between the SSE heartbeat thread and the read thread writing concurrently).
- **Fix: intermittent "service unstable / chart failed: local chart service not ready"**: fixed AI streams contaminating subsequent chart/predict requests, which caused intermittent `signature.error`, chart failures, and `/predict/pd` errors (an SSE flag lingering on a reused request object).
- This release includes Java backend changes (AI proxy / request interceptor) and rebundles `astrostudyboot.jar`; the desktop shell is updated too. Users on 2.3.0 are advised to update.
