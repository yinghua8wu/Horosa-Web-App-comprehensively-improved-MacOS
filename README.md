## Local-First / File-First / 本地优先 / 文件优先

Horosa’s working assumptions are intentionally local-first:

- provider keys stay local by default
- AIAnalysis materials, templates, bundles, history, and backups are persisted locally first
- export and restore are first-class features, not afterthoughts
- the product should remain useful even when the network is unstable

Horosa 的工作假设是明确的本地优先：

- provider key 默认留在本地
- AIAnalysis 的资料、模版、组合、历史和备份都以本地持久化为核心
- 导出与恢复是正式功能，不是附属功能
- 即使网络不稳定，产品也应保持可用

## Configuration / 配置

### Workspace Initialization / 工作区初始化

- Web 本地运行：`Horosa-Web/start_horosa_local.sh`
- 整站验证：`Horosa-Web/verify_horosa_local.sh`
- 停止本地服务：`Horosa-Web/stop_horosa_local.sh`

### Provider Configuration / Provider 配置

`AIAnalysis` 当前支持：

- OpenAI
- DeepSeek
- Anthropic
- Gemini
- OpenRouter
- Ollama
- Moonshot / Kimi
- 智谱 AI
- 硅基流动
- Groq
- xAI
- 自定义兼容接口

配置原则：

- API Key 本地保存，不要求服务端持久化
- 未定的公开支持策略保持占位，不伪造
- Provider 设置、健康诊断、模型拉取、流式分析都应能在 web/app 两端验证
- 仓库级实现入口优先参考：`PROJECT_STRUCTURE.md`、`Horosa_Desktop_Installer/README.md`、`Horosa-Web/astrostudyui/src/components/aianalysis/`

## Demo Flow / 演示流程

1. 启动本地 Horosa 运行态。
2. 打开主工作区，确认原有盘面和工具栏仍正常。
3. 进入 `AI分析`。
4. 选择 provider 预设和模型。
5. 选择案例、挂载资料或组合。
6. 发起流式分析。
7. 在历史页搜索、归档、导出。
8. 做一次本地备份与恢复验证。
