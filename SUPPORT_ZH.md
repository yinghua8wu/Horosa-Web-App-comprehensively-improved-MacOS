在发支持请求之前，建议先看：

- [README.md](README.md)
- [README_ZH.md](README_ZH.md)

## 公开后的支持模型

仓库正式公开后，建议支持路径是：

- bug：走 GitHub Issue 模板
- 功能请求：走 GitHub Issue 模板
- 使用问题和开放讨论：优先走 GitHub Discussions
- 安全问题：按 [SECURITY_ZH.md](SECURITY_ZH.md) 的私密路径处理

## 发支持请求时建议带上

- 平台：web、app，或两者都有
- 具体工作区或模块
- 如果是 AIAnalysis，带上 provider 模式和模型
- 可复现步骤
- 预期行为与实际行为
- 已脱敏的日志、截图或备份片段
- 是只在本机出现，还是多个环境都出现

## 正式公开前

在仓库完全公开之前：

- 先以仓库文档为主
