# 风水盘 全面增强计划：纳气盘法增强（A）+ 八卦阳宅法新增（B）

## Context（背景 · 目标 · 决策）

**诉求**：① 全面审计现有"纳气盘"（荀爽法）还缺哪些**好做**的功能并补上；② 读入"八卦阳宅"（倪海厦法）这套全新体系并接进 App；③ 给出覆盖**算法 / 页面 / 布局 / UI**的可直接照做的计划。

**现状（已考据）**：风水模块是 v2.1.9 **完整纯前端 React/Canvas** 实现——`components/fengshui/FengShuiMain.js`（三栏：左 `Col span=6` 四 panel 基础/纳气盘/标注/导出；右 `Col span=18` 三 tab 画布/判定/要点）+ `fengshuiEngine.js`（户型图载入、矩形框选、单元门+入户门角度、在图画门向、纳气盘 8×45° SVG 气红水蓝+数字 1-9、16 类标记 9 气/6 水/1 中立、`getSectorForPoint`/`evaluateMarker` 落点判定、运期反转、PNG/PDF 导出、撤销重做、`buildAiSnapshotText`、`saveModuleAISnapshot('fengshui')` 四同步）。两套法是**不同体系**：纳气盘＝方位气水；八卦阳宅＝人×房间方位→64 卦。

**已拍板决策（贯穿全计划）**：
1. **八卦阳宅做成风水模块内的"模式切换"**（纳气盘法 / 八卦阳宅法），**复用同一户型图上传 + 朝向输入 + canvas + 撤销**，**完整做**（成員动态定義 + 名在上位在下→64 卦 + 断语 + 应期成格 + 改运建议）。
2. **UI 一律中性技法名**「纳气盘法」「八卦阳宅法」，**不出现"荀爽""倪海厦"人名**（来源仅作 JS 注释留痕）。
3. **荀爽增强全做**：破局→危害人事映射、龙虎法灶台、移动盘、吉凶评分、缓解方案库、厨房洗菜池特例。
4. **倪海厦 64 卦断语中性重编**：保留卦象判断框架，断语用**中性专业措辞自撰重写**，去掉同性恋/愛滋/血光/夭折等极端与冒犯表述，凶吉一律加「需合命盘参看」尾注，**不照搬原文（规避版权）**。

**贯穿铁律**：① **零回归**——`techMode` 默认 `'naqi'`＝现状，纳气盘行为逐字节不变；② 纯前端（无后端/jar）；③ 暗黑双色走 `--horosa-text/-muted/-soft/--horosa-accent` 变量 + `.horosa-fengshui-*` 类前缀，不引新色板；④ AI 四同步（导出/导出设置/挂载/储存）随两模式同步；⑤ 断语**自撰、加合盘尾注、不下绝对断言、不照搬**；⑥ 改大段 JSX 后必 `npm run build:file`（jest 过 ≠ 能编译）；⑦ **实施时每处 UI/Canvas 改动必用 in-app preview（Claude Preview MCP：`preview_start/screenshot/click/inspect/eval`）实时截图核对**——尤其罗盘皮肤对齐、八卦盘渲染、卦象卡/评分环/龙虎卡的暗黑双色与排版，不能只靠"应该没问题"（详见 §十一.4）；⑧ **八卦阳宅法全程用简体中文**（卦名/家人称谓/断语/UI 文案，倪氏原文繁体一律转简；数据稿已为简体）。

## 二、架构：单引擎多模式（推荐）

**不新建并列引擎**。两法共享 90% 画布层（载图/矩形/朝向/盘旋转/`getDiskCenter`/撤销/缩放/导出），差异只在三处：盘扇区定义与上色、右栏分析、数据驱动（标记 vs 成員）。故：

- `fengshuiEngine.js` 加 `this.techMode='naqi'|'bagua'`（默认 `'naqi'`，**零回归**）+ `setTechMode(m)`（重绘+重建快照+pushHistory）；`drawAll`/`renderCompositeCanvas`/`buildAiSnapshotText`/`captureState`/`applySnapshot` 加 `techMode` 分支。
- **抽几何工具** → 新建 `fengshuiGeom.js`：把 `rotatePoint/angleInSector/formatAngle` 三个无状态纯函数移出并 export，供 `baguaCore` 复用（避免重写跨 0° 判断）。引擎从它 import。
- **八卦阳宅的算法/数据落 3 个无 DOM 新文件**（便于 jest，不塞引擎类）：`baguaCore.js`（纯函数）、`baguaData.js`（纯数据）、`naqiRules.js`（荀爽增强的纯数据/纯函数）。

**复用 vs 独有**：

| | 共享 | 纳气盘法独有 | 八卦阳宅法独有 |
|---|---|---|---|
| 左栏 | 基础(载图/矩形/朝向)、导出 | 纳气盘(透明/盘大小/盘心/运期)、标注(16类)、判定筛选 | **成員表单**(动态增删+角色推导预览) |
| 中栏 | canvas/工具栏/缩放/撤销 | `drawNaQiDisk`(8×45°气水) | `drawBaguaDisk`(8×45°铺满+成員房间高亮+卦名) |
| 右栏 | — | 判定面板(评分+破局+龙虎+移动盘) | 卦象面板(每成員一卦象卡) |
| 数据 | rect/angle/img | `markers[]` | `members[]` |

`techMode`/`members[]`/评分/破局结果纳入 `captureState/applySnapshot`（撤销覆盖模式切换与成員编辑）；切模式**不清空** markers/members（两套数据各自保留，便于来回对比）。

## 三、A · 纳气盘法增强（全做 · P0 · 纯前端）

全部落 `naqiRules.js`（纯数据+纯函数，可单测），引擎只做"取标记→调 naqiRules→塞进 ViewModel"接线。

**A1 破局→危害人事映射**：危害 = **通用层 + 卦位家人层**两段叠加。
- **通用层** `HARM_MAP`：破水(水物落气位)→感情波动/易因酒色耗财/人际是非；破气(气物落水位)→主人气运受压/事业受阻/决断力下降；双卫压气(气位上卫生间≥2)→气场受压，带 `remedyKey`。
- **卦位家人层** `PALACE_FAMILY`（后天八卦家人，破局落在哪个卦位扇区→危害该家人）：乾(西北)=父/老人/男主人、坤(西南)=母/女主人、震(东)=长子、巽(东南)=长女、坎(北)=次子(中男)、离(南)=次女(中女)、艮(东北)=三子(少男)、兑(西)=三女(少女)。破局危害文案 = 通用层 + 「不利{该扇区家人}」。例：破气在西北(乾)→「主人气运受压，尤不利父辈/男主人」；破水在东南(巽)→「感情/耗财之扰，尤应长女」。
- **接线**：vm 组装层对每个 `!ok` 标记按 `expected/actual` 推 `harmKey`，再按 `marker.sector` 查 `PALACE_FAMILY` 取家人，合成 `harm.affect`；统计气位上的卫生间≥2→`双卫压气`。判定面板每条冲突下加小字「危害：…」+「查看缓解」。快照 `[冲突清单]` 每条追 `（危害…）` + 新增 `[破局危害]` 段。

**A2 龙虎法灶台**（已细化实现）：龙虎的左右**以"灶口朝向"（下厨者面对的方向）为基准，不是门向**——这是关键，单看 sink/stove 坐标差无法定左右，必须有灶台朝向。
- **取灶口朝向**：给灶台标记加可选字段 `facingAngle`。复用现有"在图上画方向线"机制（仿 `startDrawDoor`，新增交互模式 `'drawStoveFacing'`）：从灶台拉一条短线表示灶口/操作面朝向；或在标注面板直接输入角度。**未设朝向时**判定卡显示「请先标灶口朝向」引导（不臆测）。
- **算法 `evalDragonTiger(stove, sink, facingAngle)`**（纯函数，`naqiRules.js`）：令 `facingVec=(sin θ, -cos θ)`（θ=facingAngle，屏幕坐标）、`rightVec=facingVec 顺时针转 90°`（＝下厨者右手＝白虎侧）。对 `v = sink - stove`：
  - `forward = dot(v, facingVec)`（沿灶口前后轴＝水火直冲方向）
  - `right = dot(v, rightVec)`
  - **左/右**：`right>0`＝白虎(右)侧、`right<0`＝青龙(左)侧；
  - **平行/垂直**：`|right| ≥ |forward|`＝并排(平行)、否则＝前后对冲(垂直)。
- **四格局映射**（`DT` 表）：左平行=格局不稳/左垂直=最差·水火相刑(主财损口舌)/右垂直=白虎低头·利投资进取财/右平行=最佳·大利偏财；各带 `level`(1–5) 与中性话术 + `remedyKey`。
- **UI**：判定面板「龙虎灶台」卡——需同时存在灶台(含 `facingAngle`)+ 一个**非洗菜池**水槽(见 A6)才判定，缺朝向时引导先标；level 以暖/冷色条呈现（≥4 偏暖利财 / ≤2 偏冷示警）。快照 `[龙虎灶台]` 段。
- **jest**：构造已知 `facingAngle` + 四种 sink 相对位置，断言四格局各命中（含 forward/right 边界）。

**A3 移动盘**（零新内核）：复用 `diskCenterMode='marker'`。判定面板床/书桌/沙发行加「以此为盘心」→ `setDiskCenterMode('marker')+selectMarker(id)`；新增 `probeAround(centerMarkerId)`：以该标记为盘心遍历其它门窗/水标记调 `getSectorForPoint`→返回 `[{label,sectorName,windOrWater}]`，呈现为小表「枕位纳气：东(震)·气位 ✓…」。快照 `[移动盘]` 段。

**A4 吉凶评分 0-100**：`scoreNaqi({windOk,waterOk,harms,dragonTiger})`——基准 100，每破局按 `HARM_MAP.score` 累减（破水/破气 -20），在位物 `+1/项`（封顶 10），龙虎 `±(level-3)*3`，钳 0-100。**锚点校准**：规格「95 分=一处破水」（jest 钉死）。显示：判定面板顶部评分环（`--horosa-accent`）+ 评级（≥85 吉 / 60-84 平 / <60 慎）。快照 `[吉凶评分]` 段。

**A5 缓解方案库**：`REMEDY_LIB`（break-water/break-wind/stop-one-bath/dragon-tiger-bad → 具体操作话术）。破局/龙虎结果带 `remedyKey`→判定面板折叠「缓解建议」+ 快照 `[缓解建议]` 段。

**A6 厨房洗菜池特例**：vm 组装层对 `type==='sink'` 标记，若存在 `type==='stove'` 且两者落同扇区或扇区差≤1（厨房区）→ 打 `_isKitchenSink=true`：破局统计**跳过它**，改交 A2 龙虎法处理。引擎不改 `evaluateMarker` 主体，只在组装层加归类（扇区相邻判断复用 `SECTORS` 索引环绕）。

## 四、B · 八卦阳宅法（计算内核 P1 + UI 接入 P2 · 纯前端）

**B1 卦位角度映射**（`baguaData.GUA_8` + `baguaCore.degreeToGua`）：**8 卦各占 45° 铺满 360°，无"无人带"**——直接复用纳气盘 `SECTORS` 的 8 段边界（北坎337.5~22.5 / 东北艮22.5~67.5 / 东震67.5~112.5 / 东南巽112.5~157.5 / 南离157.5~202.5 / 西南坤202.5~247.5 / 西兑247.5~292.5 / 西北乾292.5~337.5——`SECTORS[*].name` 已含卦名），`GUA_8` 只补{形神/家人/方位}。`degreeToGua(angle)` 归一后落 8 段之一，**必返某卦（不返 null）**。**复用现有朝向**：成員"所住方位"可直接选 8 方，或在 canvas 放成員落点→用现成 `getSectorForPoint` 取扇区→读卦（与纳气盘同一坐标系/盘心/旋转，零新几何）。

**B2 成員动态定義引擎**（`baguaCore.resolveMemberRole(person)`）：输入 `{sex,married,hasChildren,olderSistersUnmarried,olderBrothersUnmarried,roomDir}`→输出 `{role,benmingGua}`。**按优先级短路**（前者命中即返）：
1. `hasChildren`（本房中有自己的子女）→ 男=父(乾) / 女=母(坤)。**最高优先**（有子女者无论婚否皆判父/母）。
2. 否则 `married` → 男=夫(乾) / 女=妻(坤)。
3. 否则（未婚且无子女；**离婚/丧偶且无子女在房者亦归此支**）：
   - 女：`n = olderSistersUnmarried`；`role/benming = [长女巽, 次女离, 三女兑][n % 3]`（`n≥3` 取模循环：第 1·4 女→长女巽、2·5→次女离、3·6→三女兑）。
   - 男：`n = olderBrothersUnmarried`；`[长子震, 次子坎, 三子艮][n % 3]`（同理循环）。

**家人→本命卦**：父/夫=乾、母/妻=坤、长子=震、次子=坎、三子=艮、长女=巽、次女=离、三女=兑（即"名在上"的上卦）。**纯函数+jest**：6 条主分支各≥1 例 + 取模循环（第 4 女回巽、第 4 子回震）+ 离婚无子回未婚支 + 有子女优先于已婚 的边界。`roomDir`（所住方位）来源：成員表单 8 方 `XQSelect`，或 canvas 放成員落点→`getSectorForPoint` 读卦。

**B3 名在上位在下→64 卦**（`baguaData.HEXAGRAM_TABLE` + `baguaCore.composeHexagram`）：8×8 表（上卦=成員本命卦 × 下卦=房间方位卦 → {no,name}，标准《周易》卦序 1-64，公有无版权）。`composeHexagram(upperGua,lowerGua)`。**完整表已草拟并经程序校验（64 项齐全、卦序唯一、卦名与上下卦象逐项吻合）→ 见 `docs/八卦阳宅-64卦数据净化稿.md` §一，可直接粘贴**。jest 锚点：巽×乾=风天小畜(9)、坎×离=水火既济(63)、8 纯卦对角 + 全表 64/唯一。

**B4 64 卦断语数据**（`baguaData.HEXAGRAM_TEXTS` + `HEXAGRAM_META` · 中性重编）：每卦 `{name,xiang,marriage,career,health,advice,footer}`，另 `HEXAGRAM_META[no]={pace,tag}`（`pace` 驱动应期、`tag`∈{auspicious/mild/neutral/caution} 驱动卡片色）。**全 64 卦净化初稿已草拟完毕 → 见 `docs/八卦阳宅-64卦数据净化稿.md` §二·三，可直接粘贴**（已执行净化：删疾病名/性取向/血光/夭折等冒犯与宿命表述、健康改中性非诊断话术、凶卦改建设性+给改运方向、统一合盘尾注、自撰不照搬、无人名书名）。**上线前由领域专家（你）抽审定稿**。

**B5 应期/成格**（`baguaCore.computeTiming(hexagram,person)`）：基准窗口按卦"成势缓急"分档（`HEXAGRAM_TEXTS[*].pace` fast/mid/slow → 3天~3月/3月~3年/6月~6年）。**加速判定具体化**：八卦五行（**乾兑金 / 坎水 / 离火 / 震巽木 / 艮坤土**），若房间卦五行**生**本命卦五行、或两者**同五行比和** → `accelerated=true`，取窗口下沿（更快）；相克则取上沿（更缓）。逢三离散刻度（3天/3月/3年/6月/6年），UI 标「应期为概略区间」。jest 钉档位 + 相生/比和/相克三分支。

## 五、页面 / 布局 / UI（两模式）

**左栏**：

## 五·补、罗盘样式切换（盘面皮肤 · 可随意切换 · 纯前端）

**目标**：盘面视觉可在「绘制盘(现状)」与多张**透明罗盘图**间随意切换；皮肤是**纯视觉层**，判读/评分/卦象**始终用底层 `SECTORS` 数学**，不受皮肤影响。

**内置皮肤**（`DISK_SKINS` 表）：

| key | 名称 | 资源 | 朝向校准 northOffset | 适用 |
|---|---|---|---|---|
| `draw` | 绘制盘（默认） | 引擎实时绘制（现状） | — | 两模式；**默认＝零回归** |
| `naqi-luopan` | 纳气罗盘 | `纳气罗盘透明底.png` | 南上→约 **+180°** | 纳气盘法（图上数字气水＝南9红/北1蓝/西7红…与现状一致） |
| `bagua-24` | 二十四山阳宅图 | `透明二十四座山陽宅圖.PNG` | 南上→约 **+180°** | 八卦阳宅法（8 卦 + 24 山） |

**资源落地**：两 PNG 拷入 `Horosa-Web/astrostudyui/public/fengshui/skins/`（静态服务），引擎 `new Image()` 按需加载+缓存（transparent PNG）。

**引擎**：`fengshuiEngine` 加 `diskSkin`（默认 `'draw'`）+ `drawDiskSkin(ctx,sf,off,skin)`：取盘心+半径（复用 `getDiskCenter/getMaxDiskRadius`）→ `save(); translate(center); rotate((diskRotation+skin.northOffset)·π/180); drawImage(img, -r·scale, -r·scale, 2r·scale, 2r·scale); restore()`；受现有透明度 slider（`globalAlpha`）控制。`drawAll`：`diskSkin==='draw'` → 现状 `drawNaQiDisk/drawBaguaDisk`；否则画皮肤图（其上叠加半透明的**功能描边**：成員房间高亮 / 选中扇区 / 单元门·入户门线，保证交互与对位仍可见）。导出复合 canvas（`renderCompositeCanvas`）同样画皮肤。

**校准（关键）**：两图为南上（离/S 在顶、坎/N 在底），App 绘制盘为北上 → `northOffset≈180°`；**精确值在预览里以"单元门线对齐"目视微调后写死进 `DISK_SKINS`**（每图一次性标定 `northOffset + scale`，因图像边缘留白不同、`scale` 需各自试）。两图外环度数刻度方向（0/360 在 N＝底）与 App 角度系一致，旋转后单元门/入户门线仍叠加在皮肤上做对位参考。

**UI**：左栏模式切换 `XQSegmented` 下方加「盘面样式」`XQSelect`（三选，随模式给默认但**可随意切**）；切换即重绘。皮肤纯视觉，**判读/评分/卦象完全不变**。

**零回归**：`diskSkin` 默认 `'draw'`，不选皮肤时与现状逐像素一致。**24 山图当前作视觉皮肤**（8 卦判读照旧用 SECTORS）；若日后要做"24 山细分判读"再单列（P3+）。

- **暗黑/浅色双截**：每个新面板/卡片在两主题各截一次，确认语义色与可读性。
