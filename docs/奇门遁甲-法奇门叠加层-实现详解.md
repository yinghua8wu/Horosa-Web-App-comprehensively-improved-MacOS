## 二、关键口径（docx 校准）
- **六害排序**：`击刑 > 入墓 > 庚 > 白虎 > 门迫 > 空亡`；天干 > 一切，先解击刑天干。
- **八神归一**：kinqimen 阳遁回 `勾/雀`(标准转盘)，荀爽用白虎玄武 → 已在 `DunJiaCalc.buildCells` 的 `god:` 赋值归一 `勾→虎/雀→玄`(唯一 chokepoint，两处 buildCells 共用)，盘面/hover/八宫/化解/快照全一致；`DunJiaFaCalc.normalizeGod` 兜底。
- **化解 = 逐宫合并**（核心）：一宫可多害，化解须合起来。`buildJieHua` 按宫分组、去重 placements，于是：
  - **庚击刑**（同宫）→ 灭庚象 + 高放乙(乙庚合) + 低放巳(击刑六合) ＝ docx「乙巳」。
  - **庚单独**（非击刑）→ 只高放乙。
  - **庚入墓**（同宫）→ 灭庚象(只可移) + 高放乙 + 本宫(或对宫)冲墓。
  - **击刑用合**＝五合干(高放)+六合支(低放)；**入墓用冲**＝本宫优先(或对宫)放墓库之冲支(与空亡同写法，本宫优先)；**门迫用合**＝六合支(低放)；**白虎/庚单独**＝高放乙；**空亡填象**＝补本宫天盘干形象。
  - 通则：**先灭刑墓庚、再离虎迫空**；灭击刑可移可扔送、灭入墓只可移；**天干高放、地支低放**；时机＝本宫地支日/时(优先)或对宫地支日/时。八门化气大阵逐项不再重复「天干高放」(只在总则说一次)。
- **伏吟/反吟**（移星不变量、荀爽原文）：伏吟＝每宫天盘干≡地盘干；反吟＝天盘干≡对宫地盘干。

## 六、相关人员(生年干) + 命盘/事盘 双库（后续批：复用命盘管理，只存奇门设置）

### 6.1 相关人员 → 生年干 → 八门化气大阵
- 左栏底部 `is-wide` 多选「相关人员」(`DunJiaMain.renderInputPanel` 末)，options＝`listLocalCharts()`(命盘库)映射 `{value:cid, label:'姓名·生年干年'}`，可多人。
- **生年干**＝`birthToYearGan(birth)`(`DunJiaCalc.js` 新增；lunar-javascript `Solar.fromYmdHms(...).getLunar().getYearInGanZhiByLiChun()` 取首字，按立春分界、与奇门默认 `yearGanZhiType=2 立春交接` 一致；按出生串缓存)。
- **单源做法**：`applyFaRelatedToPan(pan)` 把 `{cid,name,yearGan}` 捕获快照 stamp 到 `pan.faRelatedPeople`(在 `recalc`/`requestNongli` 算出 pan 后、`saveQimenLiveSnapshot` 前；选人变更时重盖 `pan.faRelatedPeople`+刷模块快照+forceUpdate，不重排盘)。`computeProtect(pan)` **删占位「示本盘年干」**、改遍历 `pan.faRelatedPeople` 逐人 `addGan('生年干·姓名', yearGan)`；**未选则该类不显示**(用户 Q2=C)。
- **四同步零扩张**：人名+生年干随现有 `[八门化气大阵]` 段输出，**不新增顶层段、段表/preflight 不动**；事盘/命盘 payload 各存 `faRelatedPeople`，`restoreOptionsFromCurrentCase`/`restoreFromCurrentChart` 还原。

### 6.2 命盘/事盘 选择器(盘类，R2C2)
- `CHART_CATEGORY_OPTIONS`(`shi`/`ming`)，state `chartCategory` 默认 `shi`；选择器插在 **起局 与 月家 之间**(填 月家 is-wide 换行留出的 R2C2＝「值使下方·起局右方」)。

### 6.3 命盘数据管理（已核实 + 修漏）
- `localCharts.payload` 占星**从不写、加载忽略** → 奇门用 `payload.qimen` 零碰撞、命盘照常被占星等使用。
- **唯一漏洞已修**：`ChartAddFormComp.clickOk` 原只提交可见字段→新增命盘会丢 payload；已补透传 `this.props.fields.payload`(占星新增 payload 恒 null、行为逐字节不变)。**编辑/备注保存已保留 payload**(`upsert` 合并 `{...base,...values}`)。
- `astro.js` openDrawer `chartadd` 透传 `values.record`(无 record＝占星原行为)；`user.js newCurrentChart` honor `values.birth/zone`(占星不传→不变)，让奇门命盘 birth=起局时间而非 0 点。
