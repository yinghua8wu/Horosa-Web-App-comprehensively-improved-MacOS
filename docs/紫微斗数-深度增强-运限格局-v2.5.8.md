# 紫微斗数 深度增强：运限(八字式级联) + 格局自动识别 — v2.5.8

> 落后点补齐：星阙紫微此前只有「大限」可选，无流年/流月/流日/流时/小限的可交互运限，也无格局自动识别。
> 本次以**对标商业紫微 App 的功能广度**为目标补齐，但全部为**自有实现**（断语取公版古籍原创编排，命名/风格沿用既有 `ZiWei*`/JSON helper，不留任何第三方痕迹）。

## 一、交付内容

### 新增功能
1. **运限体系**：右栏新增「运限」Tab，**八字大运式级联横滑条**。层级结构（v3 定稿）：
   - **大限**（首行）→ **流年/小限（同一 tier，左侧竖向 toggle 互斥，只显其一）** → **流月 → 流日 → 流时**。流年与小限都是「大限内的年级层」（各 10 槽=该大限的 10 个虚岁/公历年），互斥选择；任选其一再向下展开流月/日/时。
   - **多层四化卡叠加**：详情区为**卡片栈**，每个已选层级各一张卡（大限 + 当前年级层 + 流月? + 流日? + 流时?），即「选了大限 12~21 + 流年乙巳 + 流月四月」就同时显示这 3 张卡，各带本层四化飞星与落宫；流年层额外附流曜 chips。卡片左边框按层级配色（大限金/流年蓝/小限青/流月绿/流日红/流时紫）。
   - 选中的**最深层**驱动中宫盘「流命宫」外宫的金色描边环（本命/大限态盘面**像素级不变**）。
2. **格局自动识别**：右栏新增「格局」Tab，命中格局卡片（名/类别/破格标记/短义/出处），从顶部起排。后端 `/ziwei/birth` 返回 `patterns[]`。
3. **AI 同步**：`buildZiWeiSnapshotText` 注入 `[来因宫]` + `[命中格局]` 段 → AI分析挂载 + 命盘储存均带；`aiExport.js` ziwei 预设加 `来因宫`/`命中格局` 段。

## 二、根因与坑（务必入库）

### 坑1 — 新后端路由必须重编 `astrostudyboot.jar`（gotcha #10）
`/ziwei/luck` 与 `/ziwei/birth` 的 `patterns` 改动在 `astrostudycn`，发布/本地启动都**不会自动重编** fat jar。须：
`cd astrostudysrv && mvn -o -f astrostudycn/pom.xml install -DskipTests && mvn -o -f astrostudyboot/pom.xml clean package -DskipTests`，再 `javap` 验 `ZiWeiController.luck()`/`ZiWeiLuck.build`/`ZiWeiPattern.detect` 与 `ziweige.json`/`ziweiliuchangqu.json` 已进 `BOOT-INF/lib/astrostudycn-1.0.0.jar`，最后 `stop+start_horosa_local.sh` 重启。**不重启 → /ziwei/luck 报 500/找不到**。

### 坑2 — `ZiWeiHelper.getDouJun(month, timezi)` 是「月名」查表，传地支必 NPE
后端 `getDouJun` 的 map 以**月名**（正月/二月…）为 key；`ZiWeiLuck` 起初误把**流年地支**传入 → `map.get(...)` 返回 null → `Cannot invoke "java.util.Map.get(Object)" because "map" is null`。
**修法**：斗君宫 index = `(子斗支index + 流年支index) % 12`（与 `ZiWeiChart` 构造时同法），新增 `ZiWeiLuck.doujunIdx()`，不再调 `getDouJun(地支)`。前端同理：`ZWLuckPanel` 用 `getDouJun(zidou, 流年支)`（前端 `ZiWeiHelper.getDouJun` 才是 index-add 版，与后端同名不同义——**勿混用**）。

### 坑3 — `getSmallDirectioinHouse` 女命分支算错（历史 bug）
后端 `(idx - startidx + 12) % 12` 应为 `(startidx - idx + 12) % 12`（女命也从辰/戌/丑/未起、再逆行）。小限此前无前端呈现故未暴露，本次修正。

### 坑4 — 紫微 `houses[]` 数组下标 ≠ 固定地支位置（最易翻车）
`houses[0]` 的地支随命盘而变（本测试盘 `houses[0]=巳`），**不能用 `DIZI.indexOf(地支)` 当数组下标**，否则流命环、四化落宫、小限起宫全部错位。
**铁律**：任何「地支 → houses 数组下标」必须**按地支搜索 `houses[].ganzi.charAt(1)`**（前端 `ZWLuckPanel.houseIdxByBranch`），不可用固定地支序。

### 坑5 — 运限计算放前端，避免每次点击打后端
`ZWLuckPanel` 全部纯前端算（年/月/时干支五虎五鼠遁 + 日干支连续 JDN 锚点[2026-05-18=壬辰，与 `BaZiLuckFlowPanel` 同源] + 流曜表 + `getLayerSihua`）。`/ziwei/luck` 为**服务端等价实现**（保留作 API/AI 备用），当前 UI 不依赖它。`getFlowStars`/`getLayerSihua`/`ziToHouseIndex` 在前端 `ZiWeiHelper.js`。

### 坑6 — 格局/运限面板在 ziwei tabpane 里会被 flex 居中
`ant-tabs-tabpane` 继承了居中布局，新面板根需 `display:block; align-self:stretch; overflow-y:auto`（**不要**复用 `horosa-astro-content-scroll` 的 grid 居中），否则卡片整体垂直居中而非顶部起排。

## 三、口径（已与用户确认）
- 流月起法=斗君法（流年斗君宫=正月，逐月顺行）。
- 流日=流月命宫初一顺行；流时=流日命宫子时顺行（五鼠遁日干→时干）。
- **小限**=男命顺行、女命逆行（只分男女、不分阴阳）；起宫(虚岁一岁) 寅午戌→辰、申子辰→戌、亥卯未→丑、巳酉丑→未；每12年一周；干支复用所在本命宫位（四化用本命宫干，非流年干）。
- 流曜：流年排全套（流禄/羊/陀/魁/钺/昌/曲/马）；流月及以下只各层四化+流命名。
- 来因宫：宫干==生年干（不排除子丑，统一口径）。
