## Context（背景 · 目标 · 决策）

**诉求**：一份"极其全面、可直接照着做"的紫微斗数改进计划——算法口径、UI 放在哪、怎么显示、还缺什么、如何美观且贴合现 UI 风格，全部钉死。

**资料来源**：① Obsidian 笔记 `玄哲/3.斗数`（北派四化 / 各派 / 经典·中州派深造讲义，35 篇）；② 项目现有紫微全链路（前端 14 组件 + Java `astrostudycn` + 20+ JSON + Python 自研引擎 + `docs/紫微斗数-深度增强-运限格局-v2.5.8.md`）。

**最关键的发现（经代码实测，颠覆"还缺什么"的直觉）**：
> **绝大多数"看起来缺"的技法，后端其实早已算好并落盘，只是默认的「四化盘」主视图没把它们画出来。** `ZWHouse.drawSihuaStars`（主四化盘）只渲染 `starsMain/starsAssist/starsEvil` 三组（`ZWHouse.js:220/232/239/246`）；而「三合盘」`ZWHouseSangHe` 额外渲染了 `starsOthersGood/starsOthersBad`（L244/278-289）与 `starsSmall`（L92）。后端 `ZiWeiChart.setup()` 已把三台八座/恩光天贵/龙池凤阁/红鸾天喜/天哭天虚/孤辰寡宿/华盖/蜚廉破碎/天德月德/天官天福/天厨/天巫/天月/解神/阴煞/截空旬空/台辅封诰/天才天寿 + 将前十二神(`ziweijiang.json`) + 岁前十二神(`ziweismallstars.json:taisui`) + 博士十二神 + 长生十二神 全部安进 chart；`ziweimonth.json`/`ziweitimezi.json` 也都是**填满**的（非占位）。`ziweistarlight.json` 的庙旺**含辅佐/凶煞**（左辅右弼/昌曲/禄存天马/羊陀火铃/空劫），不止 14 主星。`ziweige.json` 已 **34 格局**，含雄宿朝元/马头带箭/刑囚夹印/日月反背/火铃夹命/空劫夹命/羊陀夹命/禄逢冲破。（以上每条均经本轮 grep/head 实测核对。）

因此**本计划重心**＝把"已算未显"的显示出来（P0 纯前端、零 jar、最高性价比）＋ 新增「流派/四化表」可切换系统（P1 纯前端主体）＋ 运限流曜全层下沉、格局命中详情、流年神煞 ＋ 后端流派统一与少量真缺技法（P2 重编 jar）。

## 三、P0 — 纯前端高价值（零 jar，一次前端构建即生效）

### P0-1　主盘显示已落盘杂曜 + 十二神（头号修复）
**问题**：默认「四化盘」`ZWHouse.drawSihuaStars`（`ZWHouse.js:219-251`）只画三组，后端安好的 `starsOthersGood/starsOthersBad/starsSmall` 不显示。

**改法**（仿三合盘 `ZWHouseSangHe.js:244-289/92` 的成熟写法）：
1. `starsCount`（L220）改为计入五～六组：`starsMain+starsAssist+starsEvil + (starsOthersGood||[]).length + (starsOthersBad||[]).length`（`starsSmall` 单列小字另算，避免抢宽度）。
2. 画完 evil（L246-250）后追加两段循环：`starsOthersGood` 用 `ZWCont.ZWColor.StarOthersGoodStroke`（青）、`starsOthersBad` 用 `StarOthersBadStroke`（赤），调用既有 `this.drawStar(...)`。
3. `starsSmall`（将前/岁前/博士十二神）：仿 `ZWHouseSangHe.js:92` 用最小字号 + 灰 `StarSmallStroke`，渲染在宫格下沿单列，**默认可关**（见 P0-4）。
4. 字号/折行：杂曜数量可能多，`drawStar` 的 `avgsz`（L224）按总数缩放；超阈值时降字号或折行，**绝不溢出宫格**。
5. **行高对齐（关键细节，已核实）**：三合盘对应方法是 `drawStars`（`ZWHouseSangHe.js:243`），其星块行高 `h = sz*3 + margin*2`（L256，**预留 3 行**以容纳 Others 组）；而四化盘 `drawSihuaStars` 当前 `h = sz*2 + margin*2`（`ZWHouse.js:231`，**只 2 行**）。改造时须把四化盘的 `h` 同步提到 `sz*3`（镜像三合盘），否则追加的杂曜会与主曜/四化 badge 叠压。`starsSmall` 在三合盘是在 `drawTitle`/角位单独绘（L92），四化盘比照单列处理。

**显示风格（贴合现 UI）**：主曜大字棕金、辅曜青、凶曜赤、杂吉青、杂凶赤、十二神灰小字——颜色全部复用 `ZWConst.ZWColor.*` 既有变量，明暗自适应。tooltip 走 `this.ZWRules.RuleStars[name]`（已就绪；缺 tip 的星不弹、不报错，P1-D 补全）。

**验收**：起董鴻煦盘 → 四化盘每宫应出现杂曜/神煞（与三合盘同星集）；像素不溢出；暗色正常。

### P0-2　运限流曜下沉到全部层级

**验收**：选大限/流月/流日/流时，各层卡都出现对应流曜 chips。
