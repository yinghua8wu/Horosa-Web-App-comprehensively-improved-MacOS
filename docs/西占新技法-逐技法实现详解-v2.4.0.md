## 1. 12 分度（Dodekatemoria）

**① 概述**：每颗星在所落星座内的「十二微分度」投影点——古希腊微分技法，把 30° 星座再压缩映射一轮十二宫。

**② 算法**：`dodecaLon(lon) = norm360( floor(lon/30)*30 + (lon%30)*12 )`。即「该星座起点 + 星座内度数×12」，再归一化到 0–360。例：日 8°6′双子(=68.1°) → 60 + 8.1×12 = 157.2° = 室女 7°。

**③ 后端**：无，纯前端公式。

**④ 前端组件**：`components/astro/AstroDodeca.js`（本命「古典」tab 表格，列：本命座度 / 12分度座度）。

**⑤ AI 全链路**：
- 判读类，走本命 astro 快照。`utils/astroAiSnapshot.js`：`buildDodecaSection(chartObj)` 遍历 `AstroConst.LIST_OBJECTS`，对每星 `objAbsLon(obj)` 取绝对黄经(优先 `obj.lon`，缺则 `LIST_SIGNS.indexOf(sign)*30+signlon` 兜底)→ `dodecaLonOf` → 复用现成 `lonToSignDegree` 出「度˚座分；界」。在 `buildAstroSnapshotContent` 希腊点后 push `[12分度]`。
- **导出**：`aiExport.js` `AI_EXPORT_PRESET_SECTIONS.astrochart`/`astrochart_like` 含 `12分度`；随 astrochart 一并导出。
- **挂载**：随 `astrochart` 技法快照挂载（不是独立技法）。
- **储存**：从盘实时算，无需额外存档。

**⑥ 验证**：core 逐星到角分一致（☉15°58′♓/♄18°36′♎…）；preview 重载快照实测 日→室女7°、月17°天蝎→金牛26°。

## 2. 主宰星链（Dispositor chains）

**① 概述**：每星所落星座的「庙主星」，顺链直到落入自家星座的终极主宰（或互容成环）。

**② 算法**：星座庙主表（传统七政，按白羊→双鱼）：`[火,金,水,月,日,水,金,火,木,土,土,木]`。`rulerIdOfLon(lon)=TRAD_SIGN_RULERS[floor(norm360(lon)/30)]`。从某星起，反复取「当前星所落座的庙主」入链，遇 `ruler===自身`(落己宫)或链中已出现(互容环)即停。

**③ 后端**：无。

**④ 前端组件**：`components/astro/AstroDispositor.js`（主宰链 + 宫神星表）。

**⑤ AI 全链路**：
- `astroAiSnapshot.js` `buildDispositorSection`：用 `TRAD_SIGN_RULERS`(AstroConst 行星常量，**不引 signs.js**避免小写 key 失配)；七政各出一条链 `日：日 → 水`。push `[主宰星链]`。
- **导出/挂载/储存**：同 12 分度（随 astrochart）。

**⑥ 验证**：preview 实测 水→水(落己宫双子，庙)、月→火→金→月(火金月三星互容环正确闭合)。

## 3. 寿命格局（Hyleg / Alcocoden）

**① 概述**：希腊化寿命术——生命主(Hyleg/释放星)、寿主星(Alcocoden/给数星)、预测寿数、盘主体系(占控/家主/盘主)。

**② 算法**：引擎 `divination/lifespan/lifespanEngine.js` `runLifespan(facts, {method:'ptolemy'})`：`selectHyleg`(候选 日/月/福点/朔望/上升，按昼夜+多罗修斯否决)→ `findAlcocoden`(生命主有尊贵且相照之星)→ `computeYears`(行星年 great/mean/least + angularity 档 + 修正)→ `rulersOfLife`/`planetStates`/`medicalCrisis`。返回 `{hyleg, alcocoden:{...,predictedYears}, rulers, isDiurnal, ...}`。

**③ 后端**：无（`facts` 由 `divination/engine/chartFacts.buildFacts` 从盘派生）。

**④ 前端组件**：`components/astro/AstroLifespan.js`（本命「古典」tab 底部；取主法 ptolemy/默认）。

**⑤ AI 全链路**：
- `astroAiSnapshot.js` `buildLifespanSection`：`import buildFacts, {runLifespan}`(纯引擎、无环)；`chartObjWithFactsMaps` 非破坏补 `objectMap`(按 `o.id`)/`houseMap`(按 `h.id`)→ `buildFacts` → `runLifespan`。输出 区分/生命主/寿主星/预测寿数/盘主体系。push `[寿命格局]`。
- **坑（已记 skill/AGENTS）**：引擎产出 key/sign **全小写**(`asc`/`jupiter`/`sagittarius`，因 buildFacts 统一 `toLowerCase`)。故加 `LIFESPAN_KEY_TO_ID` 映射(小写→`AstroConst.ASC/JUPITER…`)给 `msg` 出中文；位置一律用 `lonToSignDegree(hy.lon)`(不用引擎小写 sign，否则中文名+界取不到)。
- **导出/挂载/储存**：同 12 分度（随 astrochart）。

**⑥ 验证**：core 生命主=日(Predominator)、寿主=水(Oikodespotes)；preview 实测 夜生盘·生命主上升15°射手金界第1宫·盘主体系 占控上升/家主木/盘主火（中文与 AstroLifespan 同引擎）。

## 4. 界推运（Distributions / 分配法）

**① 概述**：上升点经主限运动穿越各埃及界 → 分配星(界主星) + 该期间上升点触及的参与星 + 起止日期。

**② 算法**：后端复用既有已校验的 termdirection（界推进=主限法推上升点过界）。返回 `{dist:[{distributor, sign, participants:[], startDate, endDate}]}`。

**③ 后端**：Python `astropy/astrostudy/perpredict.py` `getDistributions()`(模块级 `from astrostudy.termdirection import TermDirection`)→ `websrv/webpredictsrv.py` `dist()` 端点 → Java `PredictiveController` `/predict/dist` 白名单路由。

**④ 前端组件**：`components/astro/AstroDistributions.js`（星运 tab，`request(ServerRoot/predict/dist, {body:chartParams(value)})`）。

**⑤ AI 全链路**（预测类，**不**入本命快照）：
- **挂载**：`AstroDistributions.js` export async `buildDistributionsSnapshotText(chartObj)`(内部同口径 fetch /predict/dist，markdown 表 + `AstroTxtMsg` 中文名，**无数据 return ''**)；`aiAnalysisContext.js` import + `regenerateChartTechniqueSnapshot` 加 `case 'distributions'`(仿 firdaria：先 `fetchChartResultForRecord` 再 await builder) + `TECHNIQUE_LABELS.distributions='星运-界推运'` + `ANALYSIS_CHART_TECHNIQUES` 加 `'distributions'`。
- **导出**：`aiExport.js` 六处登记——`AI_EXPORT_TECHNIQUES`、`AI_EXPORT_PRESET_SECTIONS.distributions=['界推运（分配法 / Distributions）']`、`predictiveLabelMap`、`predictiveMap`(domain `predictive_raw`)、`isPredictiveExportKey`、`predictiveKeys`(@4842)。抓取走刷新事件：`AstroDistributions` 监听 `horosa:refresh-module-snapshot`，`detail.module==='distributions'` 时 await builder 写回 `detail.snapshotText`。
- **储存**：从盘实时算。
- **坑**：builder 空数据若仍 push 表头 → 被判 `available=true` → `aiAnalysisContext.test`「未挂载技法须 missing」红 + `aiExport.test` 的 `getAIExportAuditMatrix` 要求每技法 6 张表(presetSections/options/extractionKind/structuredSnapshotKeys…)齐全。两处自检都已绿。

**⑥ 验证**：core Forecast→Distributions（releaser=ASC、Egyptian bounds、Classical）口径一致；build 绿 + 140 测试。

## 5. Huber 年龄推进点（Age Point）

**① 概述**：年龄点自上升点起、沿 Koch 宫顺行，每宫 6 年、72 年回归上升；落于本命星处(合相)为人生关键节点。

**② 算法**：后端 `agepoint.py`：`_koch_cusps(jd,lat,lon)` 经 `swisseph.houses_ex2(...,b'K')` 取 Koch 宫头；AP 自 ASC 起，`age//6` 定宫、宫内线性插值定黄经；与本命星同度=关键岁。返回 `{agepoint:{points:[{age,sign,signlon,house,aspectTo,aspect}]}}`。

**③ 后端**：Python `astropy/astrostudy/agepoint.py`(新建) + `perpredict.py getAgePoint()`(惰性 import) + `websrv/webpredictsrv.py agepoint()` + Java `PredictiveController /predict/agepoint`。

**④ 前端组件**：`components/astro/AstroAgePoint.js`。

**⑤ AI 全链路**：与界推运完全同构——`buildAgePointSnapshotText`(fetch /predict/agepoint，含「关键岁数」摘要)；`aiAnalysisContext` `case 'agepoint'` + label + list；`aiExport` 六处登记(`agepoint`)；`AstroAgePoint` 监听刷新事件(`detail.module==='agepoint'`)。空数据 return ''。

**⑥ 验证**：core Huber（age0=ASC、6年/宫、36岁落 DESC、72年周期）；build + 测试。

## 6. 世俗盘（Mundane / 入宫盘 + 大合相）

**① 概述**：入宫盘=太阳进入基本宫(春分/夏至/秋分/冬至)精确时刻的天宫图，世俗占星用；另附木土大合相年表(任意年段)。

**② 算法**：入宫=精确节气种子时刻起盘(确定性 astro 盘)。大合相=`compute_great_conjunctions`：扫木土带符号黄经差过零点(步长8天，二分细化50迭代)，cap 提到 ~3400 年。

**③ 后端**：`/jieqi` 精确节气种子(`fetchPreciseJieqiSeed`，强制 `lat:'23n26'` 避南半球节气互换) + `/chart` 起盘；大合相 `astropy/astrostudy/astroextra.py compute_great_conjunctions` → `websrv/webastroextrasrv.py` → Java `AstroExtraController /astroextra/greatconj`(仅 startYear/endYear 参数)。

**④ 前端组件**：`components/mundane/MundaneMain.js`（基于通用 `DivinationChartShell`；含 AGE_EPOCHS 时代纪元表 + computeAges + 主/强/落 dignity + 纯/摄/摄大 trigon 相态）。

**⑤ AI 全链路**（辅盘/事盘，**不**走前述判读段或预测 wired-technique 两条路，走第三条「事盘 payload 快照」）：
- **储存 + 挂载**：事盘存档时把格式化 astro 快照写进 `payload.aiSnapshot`：`DivinationChartShell` 加**可选** `buildAiSnapshot(chart,fields,extra)` prop(shell 保持通用)；`MundaneMain` 传 `(c,f,ex)=>[世俗入宫]段 + buildAstroSnapshotContent(c,f)`(故世俗盘 AI 自带 12分度/主宰链/寿命等全段)；`divinationCaseSave.openDivinationCaseDrawer` payload 收 `aiSnapshot`。挂载时 `extractCaseSnapshotText` 读 `payload.aiSnapshot`('ready')，不再 JSON 裸转。事盘注册 `localcases.CASE_TYPE_OPTIONS` module=`mundane`，extra(ingressTerm/Year/Moment)通用存取。
- **导出**：`aiExport.js` `auxchartMap.mundane={key:'mundane'}`、`AI_EXPORT_TECHNIQUES`、`AI_EXPORT_PRESET_SECTIONS.mundane=['世俗入宫',…astro 段]`、`getExtractorKindByExportKey('mundane')='predictive'`(走刷新事件)、`getStructuredSnapshotKeysByExportKey('mundane')=['mundane']`。`DivinationChartShell` 监听刷新事件，`detail.module===saveModule` 时用 `buildAiSnapshot` 写回。
- **坑**：mundane 用 `'predictive'` 抽取种(走刷新事件抓 detail.snapshotText)，**不能**用 `'astro'`(那会抓主盘而非入宫盘)。

**⑥ 验证**：大合相独立验到度(2000金牛22.7°/2020宝瓶0.5°/2040天秤17.9°)；入宫位置基(位置已验)。
