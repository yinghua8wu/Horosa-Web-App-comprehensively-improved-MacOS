# Horosa for macOS v1.1.7

> Developer ID distribution, offline `.pkg zip` as the primary install entry, notarized shared runtime delivery, and a hardened update-and-recovery flow

## Overview

`v1.1.7` is the release where the Horosa macOS delivery pipeline stops feeling like an internal build and starts behaving like a proper public distribution channel. This version brings the public download surface, Developer ID signing, Apple notarization, offline `.pkg zip` delivery, shared runtime packaging, update manifest flow, and recovery paths into one coherent release system.

For end users, that means a simpler message: download the offline package, install it, and open the app. For maintainers, it means a repeatable release pipeline with real validation gates.

**Release date**: 2026-03-29

## Highlights

- **Public distribution is now production-grade**: both the macOS app and the offline installer package are signed with Developer ID identities and accepted by Apple's notarization service.
- **One recommended public download**: the public install recommendation now collapses to `Horosa-Installer-macos-arm64-offline-pkg.zip`.
- **Offline installation is genuinely ready-to-open**: the installer carries the shared runtime path instead of treating a post-install network fetch as the default fallback.
- **The shared runtime is part of the signing story**: embedded Python, native extensions, runtime bundles, and nested jar-native binaries are handled as part of the notarized delivery path.
- **The in-app update model remains intact**: `horosa-latest.json`, the separate runtime release, and the app/runtime swap flow continue to work with the new signed distribution model.

## Recommended Download

Regular users should download:

- `Horosa-Installer-macos-arm64-offline-pkg.zip`

It is the best choice for:

- first-time installation
- mainland China or weak-network environments
- forwarding the package to another user or another Mac
- installs that should open immediately without a follow-up runtime download

Other `.pkg`, `.zip`, runtime, and manifest assets remain in the release because the installer and updater need them. They are not the public recommendation.

## Installation Steps

1. Download `Horosa-Installer-macos-arm64-offline-pkg.zip`
2. Extract the zip
3. Run the `.pkg` inside
4. Open `/Applications/星阙.app`
5. If macOS still blocks the app, use `Open-XingQue-Unsigned.command` from the same folder as a fallback

## Major Improvements

### Public Distribution Pipeline

- `.app` builds are signed with `Developer ID Application`, secure timestamps, and hardened runtime
- the offline `.pkg` is signed with `Developer ID Installer`
- the app and installer package are notarized independently and stapled after acceptance
- the final output is verified against Gatekeeper and stapler validation

### Shared Runtime and Offline Delivery

- the shared runtime payload is cleaned before packaging to avoid shipping development leftovers
- the embedded Python runtime repair and bundle-signing flow is stronger and more explicit
- native binaries nested inside archived runtime content are now included in the signing pipeline
- obsolete `osx-x86_64` native jars inside `astrostudyboot.jar` are removed from the Apple Silicon public path to reduce notarization risk

### Install and Recovery Experience

- offline installs are expected to be ready to launch immediately after setup
- if the offline runtime path is damaged, the app now prefers reinstallation guidance instead of silently switching to a network-based repair path
- startup now isolates the embedded Python environment more strictly so it does not accidentally import user-level third-party site-packages that can break code-signing compatibility

### Release and Maintenance Workflow

- repository-facing documentation is moving to a bilingual structure
- release notes are now versioned documents instead of one rolling note file
- the publish workflow is being aligned to read versioned release-note sources and generate GitHub release bodies from them

## Upgrade and Compatibility Notes

- public platform target: Apple Silicon (`arm64`)
- public install path: offline `.pkg zip`
- lightweight online `.pkg` and `DMG` are no longer the user-facing primary entry
- the existing in-app update model remains part of the product
