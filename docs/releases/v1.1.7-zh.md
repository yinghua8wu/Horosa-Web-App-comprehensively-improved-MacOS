# 星阙 Horosa for macOS v1.1.7

> Developer ID 正式分发、离线 `.pkg zip` 主入口、共享运行时签名与公证链路、应用内更新与恢复流程加固

## 概览

`v1.1.7` 是 Horosa macOS 分发链路的正式收口版本。这个版本不只是“再发一个安装包”，而是把公开下载、Developer ID 签名、Apple notarization、离线 `.pkg zip` 交付方式、共享运行时打包、应用内更新清单和恢复路径统一到了同一条稳定链路上。

最终结果是：普通用户现在可以直接下载离线安装包，安装后立刻打开使用；维护者也拥有了更清晰、可复用、可验收的发布流程。

**发布日期**：2026-03-29

## 本次亮点

- **正式外部分发可用**：macOS 应用与离线安装包都已接入 Developer ID 签名与 Apple notarization。
- **唯一推荐下载入口**：公开下载入口收敛为 `Horosa-Installer-macos-arm64-offline-pkg.zip`。
- **离线路径真正可用**：离线安装包内置共享运行时，不再把首次联网下载当作默认兜底。
- **共享运行时也纳入签名链**：嵌入式 Python、原生扩展、运行时 bundle 与嵌套 jar 原生库被纳入完整签名与公证处理。
- **应用内更新链路继续保留**：`horosa-latest.json`、runtime 独立 release 和桌面更新替换逻辑继续存在，并与正式签名分发兼容。

## 推荐下载

面向普通用户，请下载：

- `Horosa-Installer-macos-arm64-offline-pkg.zip`

推荐场景：

- 第一次安装 Horosa
- 中国大陆或弱网环境
- 需要把安装包发给别人
- 希望安装后直接可用，不再额外联网准备运行时

不建议普通用户手动处理其他 `.pkg`、`.zip`、runtime 或 manifest 资产。它们主要给安装器、自动更新器和发布系统使用。

## 安装步骤

1. 下载 `Horosa-Installer-macos-arm64-offline-pkg.zip`
2. 解压 zip
3. 双击里面的 `.pkg`
4. 安装完成后，从 `/Applications/星阙.app` 打开
5. 如果系统仍然拦截，再运行同目录的 `Open-XingQue-Unsigned.command` 作为兜底

## 主要改进

### 公开分发链路

- `.app` 使用 `Developer ID Application` 签名、secure timestamp 与 hardened runtime
- 离线 `.pkg` 使用 `Developer ID Installer` 签名
- 应用与安装包分别提交 Apple notarization，并完成 `staple`
- 构建结果经过 Gatekeeper 与 stapler 验证

### 共享运行时与离线路径

- 共享 runtime 打包前会清理不必要的开发残留文件
- 嵌入式 Python runtime 的 bundle 签名与修复流程得到加固
- 嵌套在运行时归档中的原生二进制也被纳入签名流程
- `astrostudyboot.jar` 中与当前分发目标无关的旧 `osx-x86_64` 原生包被移除，减少 notarization 风险

### 安装与恢复体验

- 离线安装完成后，应直接具备启动条件
- 如果共享运行时已损坏，恢复路径优先提示重新安装离线包
- 启动阶段对嵌入式 Python 运行环境做了更严格的隔离，避免误读用户目录里的第三方 site-packages 导致签名冲突

### 发布与维护

- 仓库文档升级为中英双语结构
- Release 文档改为版本化维护
- 发布脚本开始转向从版本化 release 文档源生成 GitHub Release 正文

## 升级与兼容

- 平台目标：Apple Silicon (`arm64`)
- 兼容的公开安装方式：离线 `.pkg zip`
- 轻量在线 `.pkg` 与 `DMG` 不再作为普通用户公开入口
- 现有应用内更新链路继续保留
